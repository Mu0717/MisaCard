# 🚀 MisaCard 生产环境部署文档

本文档详细记录了如何将 MisaCard 项目部署到你的云服务器上。我们已经为你更新了项目的架构，引入了优雅的 **Caddy 反向代理容器**，这意味着你**完全不需要**在服务器上安装 Nginx，也不用手动申请 SSL 证书。只要运行一条脚本，HTTPS 将会全自动为你搞定！

## ✅ 2. 前置网络准备（极其重要）

在你登录服务器进行任何部署前，请务必先完成以下两步（否则 HTTPS 证书一定会因验证不通而申请失败）：

1. **配置域名解析 (DNS)**：
   前往你的域名服务提供商的控制台（如 Cloudflare, 阿里云等）。
   在域名 `howard0717.com` 的解析设置页，**添加一条 A 记录**：
   - 主机记录/前缀填：`card`
   - 记录值/目标IP填：`104.154.247.43`

2. **开放云服务器端口（安全组）**：
   登录管理你那台 `104.154.247.43` 服务器的云控制台，找到其对应的“安全组”或“防火墙”，**确保放行以下三个协议/端口**：
   - **TCP 80**  (用于 HTTP 访问以及 Caddy 自动申请验证证书的核心端口)
   - **TCP 443** (用于 HTTPS 访问加密的端口)
   - **TCP 22**  (你平时用来 SSH 远程连接的端口，一般默认已开)

## 🛠 3. 服务端环境部署与拉取

该套件对系统侵入极低，服务器只需具备三个最基础的工具：`git`、`docker`、`docker-compose`。

1. **通过 SSH 终端连接这台新服务器**：
   ```bash
   ssh root@104.154.247.43
   ```

2. **克隆你的项目代码**：
   *(如果这是你第一次在服务器操作，请先 Clone 仓库。如果之前跑过旧版就跳过 clone，直接进目录)*
   ```bash
   git clone <你仓库的地址>
   cd MisaCard
   ```
   > 这里的代码中，我们已经为你准备好了最新的 `Caddyfile`、升级版的 `docker-compose.yml` 和支持编排服务的 `redeploy.sh`。

---

## 🚀 4. 执行一键上线

1. **赋予自动化构建脚本运行权限**（防止遇到 permission denied）：
   ```bash
   chmod +x redeploy.sh
   ```

2. **跑起来吧**：
   ```bash
   ./redeploy.sh
   ```

   **脚本会按顺序全自动执行以下操作**：
   1) 从 Git 仓库拉取最新代码变更。
   2) 优雅地停止之前仍在运行的老环境。
   3) 重新构建你的 FastAPI 业务后端镜像。
   4) 根据 `docker-compose.yml` 去编排，在同一个 Docker 内网下启动 `misacard-manager` 与 `caddy` 容器服务。

3. **检验成果**：
   当终端日志打出 `==> 部署完成！` 时，Caddy 容器就已经在后台默默向 Let's Encrypt 签发 `card.howard0717.com` 的证书。
   **只需耐心等待大约 10 秒钟左右**，接着就可以在你的电脑浏览器里试着访问：

   **网址** 👉 `https://card.howard0717.com`

   若能看到页面且浏览器显示“小绿锁”（安全连接提示），那么恭喜，MisaCard 已经完美且安全地部署在生产环境了！

## 💡 5. 日常排障与维护指南

日后维护或是遇到奇怪的问题，只需 cd 进 `MisaCard` 项目目录，运行以下常用的管理命令：

**1. 实时查看项目日志（排错神器）**
```bash
# 所有容器混看的总日志
docker-compose logs -f

# 仅查看后端业务的日志 (例如发现页面报错 500 返回排查断点)
docker-compose logs -f misacard-manager

# 仅查看代理层 Caddy 的日志 (例如看证书因为什么鬼原因没发下来)
docker-compose logs -f caddy
```

**2. 停止并关闭整体环境**
```bash
docker-compose down
```

**3. 热重启后端业务逻辑**（比如你手动进容器或挂载卷改了环境配置）
```bash
docker-compose restart misacard-manager
```
